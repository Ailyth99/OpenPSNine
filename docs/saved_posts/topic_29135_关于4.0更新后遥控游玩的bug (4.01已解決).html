关于4.0更新后遥控游玩的bug (4.01已解決)

tayhtac 2016-10-05 09:04编辑     21评论
不知道有没有火星，自己遇到了所以也放出来，希望能帮到有需要的同志们

说重点：

PS4更新4.0及Remote Play官方软件更新至1.5版本后，无论是Win10还是Mac，都会遇到无法连接，网速过慢，或在连接后黑屏提示断线的问题。具体的原因其实不是网络问题，而是新remote play的软件在连接时会检查HDMI协议 （HDMI HANDSHAKE）。具体不知道是remote play的bug还是4.0的bug，据说当时4.0 beta的时候就有人反映过这个问题。

在家里电视机打开后（是否需要切换到PS4输入源不清楚），便可以成功连上remote play。无论是外网还是内网连接。

在家里人帮忙打开电视后，在公司网络测试，成功连接并稳定游玩。

所以遇到同样问题的基友们，这不是你网络环境的锅，这只是sony的锅。

然後骚尼已经把锅在4.01解决了

HDMI handshake = The exchange of encrypted keys between the source / player and the LCD and HDMI Distribution equipment. Each device features a unique set of keys which need to be accepted by the display and source before video/audio is displayed. http://site.hdtvsupply.com/hdcp1.html
ps4启动的时候，想要通过HDMI在电视上播放画面，必须经过这个handshake过程。如果电视机关掉，那这个过程就走不了。 照道理遥控游玩是不需要这个的，所以这个bug才让人费解。至于为什么显示器就没有这个问题，我也不知道啊
